[조건]
- 2차원 String 배열로 게시물을 관리

- 게시물의 총 수는 100개로 한정
     String[][] = new String[100][4]; // 100개의 게시물, 4개의 기본정보 번호, 제목, 내용, 글쓴이

- 저장 형태
     { 
       {"1", "제목", "내용", "글쓴이"}, 
       {"2", "제목", "내용", "글쓴이"}, 
       { ... }, { ... }, { ... }, { ... }, { ... }, ...., 
       null, null, null 
     }

- 번호는 자동 증가 되어야함 (0)
- 삭제는 null로 저장 (0)
- 번호를 내림차순 정렬
- 배운 내용으로만 작성할 것 (다른 API 사용 불가) (0)





- 인덱스 앞에서부터 null을 없앨 것... 
3번게시물 2번게시물 1번게시물 -> 2번 게시물 삭제 -> 3번게시물, null, 1번 게시물 , null, null
-> 3번 게시물, 4번 게시물, 1번 게시물, null, null, null ~~~~
